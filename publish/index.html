JavaScript

JavaScript — это кросс-платформенный, объектно-ориентированный скриптовый язык, являющийся небольшим и легковесным. Внутри среды исполнения JavaScript может быть связан с объектами данной среды и предоставлять программный контроль над ними.

JavaScript

Ядро JavaScript расширяется для различных целей путем добавления в него новых объектов, определяющих API для взаимодействия с внешним миром, например:

  • JavaScript на стороне клиента расширяет ядро языка, предоставляя объекты для контроля браузера (Browser Object Model (BOM)), а также непосредственно Document Object Model (DOM), о которых мы поговорим далее.
  • JavaScript на стороне сервера расширяет ядро языка, предоставляя объекты обращения к файловой системе, обмена информацией по протоколу HTTP и т.п.
  • JavaScript в базах данных в свою очередь расширен объектами доступа к элементам этих баз данных.

Browser Object Model

Document Object Model

Объектная модель документа (Document Object Model, DOM) - это фундаментальный прикладной программный интерфейс, обеспечивающий возможность работы с содержимым HTML и XML-документов.
Прежде всего, следует понимать, что вложенные элементы HTML или XML-документов представлены в виде дерева объектов DOM. Древовидное представление HTML-документа содержит узлы, представляющие элементы или теги, такие как <body> и <p>, и узлы, представляющие строки текста.

DOM as tree

<html>
  <head>
     <title>Пример документа</title>
  </head>
  <body>
     <h1>Это HTML-документ</h1>
     <p>Пример <i>простого</i> текста.</p>
  </body>
</html>

Получение элементов на странице

1 - По уникальному идентификатру id:

<div class="my-block">
    <input name="el_name" class="my-input"/>
    <input id="myElementId" type="email" class="my-input"/>
</div>


var input = document.getElementById("myElementId");

Получение элементов на странице

2 - По атрибуту name:

<div class="my-block">
    <input name="el_name" class="my-input"/>
    <input id="myElementId" type="email" class="my-input"/>
</div>


var inputs = document.getElementsByName("el_name"); // NodeList

Получение элементов на странице

3 - По тегу:

<div class="my-block">
	<input name="el_name" class="my-input"/>
	<input id="myElementId" type="email" class="my-input"/>
</div>


var inputs= document.getElementsByTagName("input"); // NodeList
var div= document.getElementsByTagName("div")[0]; // First (and one in current situation)
var theSameInputs= div.getElementsByTagName("input");

Получение элементов на странице

4 - По классу

<div class="my-block">
	<input name="el_name" class="my-input"/>
	<input id="myElementId" type="email" class="my-input"/>
</div>


var inputs= document.getElementsByClassName("my-input"); // NodeList
var theSameInputs= div.getElementsByClassName("my-input"); // Available for Elements

Получение элементов на странице

5 - По CSS селектору.

Метод querySelector вернет первый соответствующий селектору элемент.

<div class="my-block">
    <input name="el_name" class="my-input"/>
    <input id="myElementId" type="email" class="my-input"/>
</div>


var emailInput = document.querySelector('input[type="email"]'); // Node
var sameEmailInput = div.querySelector('input[type="email"]'); // Available for Elements

Получение элементов на странице

5.1 - По CSS селектору.

Метод querySelectorAll вернет все соответствующие селектору элементы.

<div class="my-block">
    <input name="el_name" class="my-input"/>
    <input id="myElementId" type="email" class="my-input"/>
</div>


var inputs = document.querySelectorAll('div > .my-input'); // NodeList
var sameInputs = div.querySelectorAll('input.my-input'); // Available for Elements

Немного о коллекциях NodeList

Как уже было замечено, в тех случаях, когда результатом может являтся более одного элемента, возвращается коллекция элеменнтов NodeList. NodeList это псевдо-массив и не содержит в цепочке прототипов прототип Array.prototype, поэтому не получится напрямую вызывать методы массивов на объектах NodeList. 

var nodes= document.getElementsByTagName("div");
 
for(let i=0; i < nodes.length; ++i) {
    let node = nodes[i];
    // ...
}
 
nodes.forEach((node)=>{ /*...*/ }); // TypeError: nodes.forEach is not a function

Вызов методов Array на псевдо-массивах

Выход есть, и возможен он благодаря функциям call и apply из Function.prototype. Мы можем вызывать функции прототипа Array в контексте объектов NodeList. Также бывает удобным и преобразовавать NodeList в Array при помощи все того же способа - вызова в контексте и метода slice[Array.prototype]. 

var nodes= document.getElementsByTagName("div");
Array.prototype.call(nodes, (node)=>{ /*...*/ });
 
var nodesArray = Array.prototype.slice.call(nodes); // convert NodeList to Array
nodesArray.forEach((node)=>{ /*...*/ });

Такой "трюк" можно использовать и на других псевдо-массивах, например на arguments.

Доступ к окружению элемнтов страницы

Помимо описанных ранее методов для получения произвольного элемента в дереве сам интерфейс (прототип) Node содержит динамические ссылки на все "окружение" узла дерева: родительский элемент (parentNode), коллекцию дочерних элементов (childNodes), ссылки на первый/последний дочерний элемент (firstChild/lastChild), ссылки на соседей в родительском элементе (previousSibling / nextSibling). Создание элемента

Для того, чтобы создать новый элемент, не привязанный к дереву, используется метод createElement [Document.prototype]. 

var newDiv = document.createElement("div");

Клонирование элемента

Для того, чтобы создать копию существуюего узла, используется метод cloneNode [Node.prototype]. 

var dupNode = node.cloneNode(deep);

Изменеие иерархии элементов

  • appendChild(node) [Node] - Вставляет node как последний дочерний узел данного элемента.
  • insertBefore(newNode, referenceNode) [Node] - Вставляет newNode непосредственно перед referenceNode, потомком данного элемента.
  • replaceChild(newNode, oldNode) [Node] - Заменяет одного потомка (newNode) из существующего на второй (oldNode).
  • removeChild(node) [Node] - Удаляет дочерний элемент node из DOM. Возвращает удаленный элемент.

См. Node API

Пример

var container = document.querySelector('div.container');
 
var stub = container.querySelector('.empty-stub');
stub && container.removeChild(stub);
 
var row = document.createElement('div');
row.textContent = (new Date()).toString();
 
container.appendChild(row);

Главное правило DOM манипуляций: cократить количество изменений live DOM на столько, на сколько возможно.

Чем меньше мы меняем live DOM (отслеживаемое дерево), тем меньше мы заставляем браузер пересчитывать позиции компонентов, обновлять CSSOM (объектная модель CSS), перерерисовывать все, что было затронуто...

Чем больше мы следуем этому правилу, тем луше производительность наших веб-страниц.

Модификация элемента

Для того, чтобы изменять содержимое элемента, предусмотренны два свойства - innerHtml и textContent

  • innerHtml [Element.prototype] - устанавливает или получает разметку дочерних элементов. При этом значение, которое мы выставляем, анализируется браузером.
  • textContent [Node.prototype] - позволяет задавать или получать текстовое содержимое элемента. При этом значение трактуется как текст, все спецсимволы автоматически экранируются.

Модификация элемента

Доступ к аттрибутам элемента осуществляется посредством методов:

  • hasAttribute(attrName)[Element.prototype]
  • getAttribute(attrName)[Element.prototype]
  • setAttribute(attrName, attrValue)[Element.prototype]

Обратите внимание на то, что для метода setAttribute оба параметра обязательны. Т.е., если мы просто хотим добавить новый аттрибут к элементу, мы должны явно указать для него пустое значение:
				
element.setAttribute("disabled", "");

Модификация списка CSS-классов элемента

Способов доступа к списку CSS-классов существует несколько:

  • параметр className [Element.prototype], сязанный с атрибутом "class".
  • динамическая коллекция classList [Element.prototype] (о ней поговорим далее)

Формально доступиться к списку классов можно и уже известным способом: методами setAttribute/getAttribute по имени "class".

Коллекция classList

Element.classList - это еще один псевдомассив, на этот раз реализующий прототип DOMTokenList. Здесь у нас в наличии чуть больше методов, чем мы видили в псевдомассивах ранее. С решением 95% задач можно справиться не уходя за рамки данного прототипа. Среди основных методов можно выделить:

  • add(className) - добавляет (если не добавлен) класс с именем className
  • remove(className) - удаляет (если есть) класс с именем className
  • contains(className) - проверка на наличие класса с именем className
  • toggle(className) - добавляет если нету и удалет если есть класс с именем className

Модификация стилей элемента

Для получения и установки инлайновых стилей используеться свойство style [HTMLElement]. Это свойство является экземпляром прототипа CSSStyleDeclaration. CSSStyleDeclaration - не псевдо-массив, а несколько более сложная структура, это хранилище ключ-значение с динамическим подсчетом их кол-ва (доступно свойство length). В качестве ключа используется название css опции в CamelCase. Т.е.:

  • для доступа к свойству color используется такой же ключ color
  • а для доступа к свойству margin-top ключ будет marginTop

В качестве значения используется строка, соответствующая тем же правилам, что и оригинальное значение CSS-свойства.

Модификация стилей элемента

Рассмотрим следующий пример: 

const MAX = parseInt('fff', 16);
function randomize() {
  let el = document.querySelector('.content');
 
  let randomColor = Math.floor(Math.random() * MAX);
  let bgColor = '#' + Number(randomColor).toString(16);
  let fgColor = '#' + Number(MAX - randomColor).toString(16);
 
  el.style.backgroundColor = bgColor;
  el.style.color = fgColor;
}

Здесь мы устанавливаем фон и цвет текста в случайные контрастные цвета.
