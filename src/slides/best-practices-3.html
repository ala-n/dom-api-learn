3. Clean Code

По статистике 80% времени программист читает код. Треть этого времени, как правило, он не лестно вспоминает тех, кто написал этот код, так что ему его приходится читать 80% времени :).

Поэтому пишите чисто. Код должен быть красивым и логичным.
